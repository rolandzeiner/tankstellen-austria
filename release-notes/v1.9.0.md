# v1.9.0 — Platinum baseline parity

Pre-release on `dev`. Brings tankstellen-austria up to the same Platinum
baseline that ladestellen 0.3.1, linz-linien 0.5.0, nextbike 1.2.0, and
wiener-linien 1.4.0 ship.

## Privacy fix (BLOCKING)

- **`dynamic_entity` device_tracker entity_id removed** from sensor
  `extra_state_attributes` and from the diagnostics envelope. The bound
  `device_tracker.*` id was previously bleeding into the recorder + the
  frontend state. UX preserved via a new `dynamic_tracker_label`
  attribute carrying only the user-chosen friendly name (no entity_id).
- New `station_display_name` attribute (locale-agnostic, sourced from
  the entry title) so the card stops regex-stripping localised
  `friendly_name` strings.
- `dynamic_entity` added to the diagnostics `TO_REDACT` set; new
  sentinel-redaction grep test guards against future fields slipping
  through unredacted.

## CI / tooling

- **Python 3.13** (no change from 3.12 → 3.13: bumped from 3.12 to 3.14
  during this cycle; rolled back to 3.13 after CI verified that HA
  core's transitive deps — pydantic-core / orjson / Pillow — still
  ship pyo3 0.24 without the `abi3` feature, so 3.14 source-builds
  fail. `PYO3_USE_ABI3_FORWARD_COMPATIBILITY=1` is a no-op for them.
  Bump again once `pip show pydantic_core orjson Pillow` shows
  pyo3 0.25+ on a fresh HA release.) Companion 3.12 → 3.13 was already
  in place across ladestellen, linz-linien, nextbike, wiener-linien.
- `requirements_test.txt`: pinned floors for PHCC, pytest-asyncio,
  pytest-cov, syrupy, mypy, ruff, plus the `pycares<5` cap for the
  aiodns 3.5 / pycares 5 `ares_query_a_result` rename.
- `pyproject.toml`: ruff `target-version = "py313"`.
- `tsconfig.json`: `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes`
  enabled; downstream narrowing applied across sparkline / best-refuel
  / math / localize / card.
- `.github/workflows/validate.yml`: `no-dev-fixture` first-job guard,
  Perl backtick scanner over Lit `css\`/html\`` templates, `npx tsc
  --noEmit` step before `npm run build`.
- New `.github/workflows/codeql.yml`: python + js/ts matrix, paths-ignore
  for the bundled card and `node_modules`.

## Refactors

- **`card_registration.py` extracted** — HA 2026.2-compatible duck-typed
  storage probe (`resource_mode` ≥ 2026.2 OR `mode` ≤ 2026.1, fail
  closed), `LOVELACE_DATA` HassKey lookup with `ImportError` fallback,
  `cast("ResourceStorageCollection", ...)` at every storage-only
  mutation site, 60-tick × 5 s capped retry loop with one WARNING on
  timeout. `JSModuleRegistration.async_unregister()` runs from
  `async_remove_entry` gated on the LAST entry remaining.
- `__init__.py`: switched to canonical `homeassistant.components.websocket_api`
  submodule imports — drops three `# type: ignore[attr-defined]`. WS
  deregister-acknowledgement comment block added.
- `async_setup_entry` reorder: `coordinator.async_setup()` +
  `entry.async_on_unload(coordinator.async_teardown)` now run AFTER
  `async_config_entry_first_refresh()` succeeds (was: before — leaked
  the device_tracker listener on a half-init that raised
  `ConfigEntryNotReady`).
- `const.py`: `CARD_VERSION = INTEGRATION_VERSION` (no more hardcoded
  literal). New `ATTRIBUTION = "Datenquelle: E-Control"` constant.
- `sensor.py`: `_attr_attribution = ATTRIBUTION` on the base entity.
- `diagnostics.py`: top-level `attribution`; coordinator block now
  surfaces `last_exception: repr(coordinator.last_exception)`. Dropped
  the bespoke `unique_id_pattern` field.

## Card

- **`<ha-form>` editor migration** — top-level schema-driven form for
  the static sections (entities, display, history, cars, payment,
  branding). Tab labels, payment-filter chips, and the cars roster
  stay bespoke below the form (data-driven row lists).
  `_computeLabel` runs the canonical three-step chain (HA core →
  `editor.<field>` → raw); `_computeHelper` reads `<name>_helper`
  with undefined on miss. Every expandable carries `flatten: true`
  (declared explicitly in `HaFormExpandableSchema`).
- **`shared-render.ts` extracted** — `checkCardVersionWS`,
  `renderVersionBanner`, `reloadAfterCacheWipe`. Banner helper
  supports tankstellen's stuck-state branch via a `stuck` flag.
- **`utils.ts` extracted** — `safeHttpsUri()` trust-boundary guard
  applied at every `<a href>` site (E-Control footer link, per-station
  map links). Centralised `E_CONTROL_HOMEPAGE` and `E_CONTROL_LOGO_URL`
  constants.
- Deleted dead `action-handler-directive.ts` (declared but never wired);
  `tap_action`/`hold_action`/`double_tap_action` removed from the
  config interface.
- Card consumes the new `dynamic_tracker_label` and `station_display_name`
  attributes; no longer does `friendly_name` stripping or `hass.states[
  trackerId]` lookups.

  `MOCK_STATION_2`, `make_entry()`. Per-module copies collapsed.
