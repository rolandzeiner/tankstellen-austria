# v1.9.1 — Test stack on Python 3.14, drift cleanup, radius slider removed

Patch release on top of v1.9.0. No user-visible feature changes; the
config-flow surface drops one cosmetic field, the rest is internal
hygiene.

## User-facing fix

- **Radius slider removed from config flow (#31)**. The map picker had
  a draggable radius circle that was never wired up — the value was
  shown but never persisted, never sent to the E-Control API, and
  never used to filter results. Reporters saw the slider reset to
  5000 m on every reopen and a station list that ignored their
  picks. The slider is gone; the map picker stays for choosing
  lat/lng, which is all it ever actually controlled. The
  E-Control Spritpreisrechner endpoint applies its own
  server-side nearby-search window and doesn't accept a radius
  parameter, so adding a working slider would require client-side
  filtering — out of scope for this patch.

## CI / test stack

- **Python 3.14** (was 3.13). HA core 2026.5.0 requires Python ≥3.14.2,
  and PHACC ≥0.13.317 wraps that HA. The earlier pyo3 0.24 blocker
  (CI failure 2026-05-01 that forced a rollback to 3.13) is gone:
  orjson 3.11.8, Pillow 12.2.0, and pydantic-core 2.46.2 all ship
  cp314 wheels. Verified locally in a fresh Python 3.14.4 venv
  against the full PHACC 0.13.329 dep tree — 82 tests pass,
  coverage 92.96%, mypy --strict clean, ruff clean.
- **Test floors bumped in lockstep** — PHACC's hard pins (`==`)
  govern, so all four had to move together:
  - `pytest-homeassistant-custom-component` 0.13.311 → 0.13.329
  - `pytest-cov` 7.0.0 → 7.1.0
  - `syrupy` 5.0.0 → 5.1.0
  - HA core (test-wrapped) 2026.2.3 → 2026.5.0
- **mypy** floor 1.20.2 → 2.0.0. Major bump verified clean against the
  integration source (zero new findings under `--strict`). PHACC
  doesn't pin mypy, so it's an independent decision.
- **Subsumes Dependabot PRs #28, #29, #30** which couldn't merge in
  isolation because each one moved one of the three pinned deps
  past PHACC's transitive constraint. Dropping them all together
  resolves the constraint cleanly.

## Portfolio drift cleanup

Audited against the scaffolding skills (`ha-integration-platinum`,
`ha-lovelace-card`, `ha-portfolio-design`) and applied four small
single-file alignments. None change runtime behaviour for valid
configs.

- `config_flow.py`: added `MINOR_VERSION = 1` next to `VERSION = 1`
  (Platinum: additive `entry.data` changes can bump `MINOR_VERSION`
  without forcing a full migration). Mirrors `ladestellen-austria`.
- `const.py` + `config_flow.py`: extracted `MIN_POLL_MINUTES` /
  `MAX_POLL_MINUTES` from the `NumberSelector(min=10, max=720)`
  literals so the form and any future migration logic share one
  source of truth.
- `tankstellen-austria-card.ts`: `setConfig` now throws on shape
  errors (non-object config, or `entities` field that is neither
  string nor array). Card skill: silent degradation hides typos
  behind the empty-state — HA should surface a proper error card.
- `editor.ts`: dropped the `if (!this.hass)` early-return render
  gate. Card skill: gating on `!hass` leaves the editor stuck
  forever when HA's `hass` property arrives a tick late
  (Lovelace `_fetchConfig` promise chain disruption); `_config`
  is already initialised to a valid minimum and the bespoke
  sections gate on hass internally.

## Strict-mode tightening (now that Python 3.14 is unblocked)

- **`pyproject.toml` ruff `target-version`** 3.13 → 3.14, in lockstep
  with the CI Python pin. Prevents silent false negatives where ruff
  was linting py3.13 rules against a 3.14 runtime.
- **`tsconfig.json`** picks up the seven flags from `typescript-strict`
  §11 baseline that were previously omitted: `noImplicitOverride`,
  `noUnusedLocals`, `useUnknownInCatchVariables`, `isolatedModules`,
  `verbatimModuleSyntax`, `esModuleInterop`,
  `forceConsistentCasingInFileNames`. Surfaced 9 real findings, all
  surgical:
  - Eight Lit lifecycle methods (`shouldUpdate`, `disconnectedCallback`,
    `updated`, `render`, `static styles`) gained their `override`
    keyword across `tankstellen-austria-card.ts` and `editor.ts` —
    catches future drift if a base-class signature changes.
  - One mixed type/value import in `sparkline.ts` (`TemplateResult`)
    split into `import type` per `verbatimModuleSyntax`.

## Frontend test stability

- **`best-refuel.test.ts` flake fixed.** The "buildHourlyEnvelope —
  duration-weighted band" test was failing on `dev` whenever the
  suite ran on any weekday other than Monday. Root cause: the
  fixture anchored its 28-day window against `deterministicNow()`
  (most-recent past Monday at midnight) but the production code
  reads `Date.now()` directly inside `walkChunks` to terminate the
  trailing segment from the last fixture point. The gap between
  fixture-now and real-now leaked a CHEAP-priced trailing segment
  through every hour-of-day bucket after 23:00 — by Thursday three
  CHEAP chunks had landed in hour 14, dropping the p10 envelope
  for that hour to CHEAP. Fix: pin `Date.now()` to the fixture
  anchor with `vi.setSystemTime(deterministicNow())` in `beforeAll`,
  restore real timers in `afterAll`. No production-code change —
  the implementation was correct; the test wasn't isolated from
  wall-clock time.
