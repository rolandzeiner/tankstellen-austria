# v1.9.2 — Distance, nearest-first sorting, smarter navigation links

Added the as-the-crow-flies distance (Luftlinie) to every station — shown
on the card, sortable, and paired with navigation links that open the
right maps app for your device. No breaking changes.

## Added

- **Distance per station (#61).** Each station now carries a `distance_m`
  attribute — the straight-line distance (metres) from your configured
  location, or from the tracked device's live position in dynamic mode.
  Thanks @Michaela0302 for the suggestion!
- **Distance on the card (#61).** The value renders as a small caption
  under the map-pin ("850 m", "12,3 km"), tucked beneath the icon so rows
  stay compact. Controlled by the new **Show distance** switch in the card
  editor — on by default for newly added cards, off on existing cards so
  they look unchanged until you enable it.
- **Sort by distance (#61).** A second editor switch orders the station
  list nearest-first instead of cheapest-first — also suggested by
  @Michaela0302.
- **Device-aware navigation links (#64).** The map pin now opens Apple
  Maps on iPhone/iPad, the maps-app chooser on Android, and Google Maps on
  the desktop. A new **Navigation app** dropdown in the card editor forces
  Google Maps or Apple Maps everywhere instead. Thanks @RookieIVG for the
  idea!

## Fixed

- Station rows no longer grow taller when the distance caption is shown —
  the map pin scales down so the card height stays put.
- Sparkline noon markers stay at 12:00 across DST transitions.
- Saving the integration's options no longer reloads the config entry
  twice.

## Changed

- Temporary API failures now back off exponentially instead of retrying at
  full cadence, easing load on E-Control during outages.
- The card registers an entity suggestion for Home Assistant 2026.6's
  entity-first card picker.

## Compatibility

- Home Assistant: ≥ 2025.1.0
- HACS: ≥ 2.0.0
